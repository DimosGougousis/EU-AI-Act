Agent 1 â€” ClassifyBot Risk Classification

Automatically classifies AI systems against EU AI Act risk tiers, citing relevant articles and annexes. Triggered when a new AI system is registered or an existing system description changes.

What it does

ClassifyBot takes a structured description of an AI system (purpose, inputs, outputs, decision role, affected users) and runs a multi-step classification workflow:

1
Prohibited AI Check

Screens against Article 5 prohibited practices list. Flags any match for immediate escalation.

2
Annex III Matching

Compares system purpose against each Annex III category (8 domains, 18+ sub-items). Returns matching categories with citation.

3
Exemption Check

If Annex III match found, checks Recital 58 fraud exemption and other carve-outs. Evaluates "sole purpose" criterion.

4
Profiling Escalation

Even if no direct Annex III match, checks if system profiles individuals in a manner that substantially influences Annex III decisions.

5
Classification Report

Generates structured JSON classification with risk tier, legal basis, obligations summary, and recommended next steps.

def classify_ai_system(system_description: dict) -> dict:
    messages = [{
        "role": "user",
        "content": f"Classify this AI system under EU AI Act (2024/1689): {system_description}"
    }]

    while True:
        response = client.messages.create(
            model="gpt-4o",
            max_tokens=4096,
            tools=tools,
            messages=messages,
            system="You are an EU AI Act classification expert. "
                   "Systematically classify AI systems using the four-tier "
                   "risk framework. Always cite specific articles and annexes."
        )

        if response.stop_reason == "end_turn":
            return response.content[0].text

        # Process tool calls and continue agentic loop
        tool_results = process_tool_calls(response.content)
        messages.append({"role": "assistant", "content": response.content})
        messages.append({"role": "user", "content": tool_results})

Agent 2 â€” DocDraftAgent Documentation

Generates Annex IV technical documentation drafts from model metadata, training records, and performance metrics. Reduces documentation effort from weeks to hours.

What it does

DocDraftAgent connects to FinPulse's internal systems (MLflow model registry, data catalog, CI/CD pipeline) and pulls structured metadata to auto-generate a compliant Annex IV technical documentation draft for human review and sign-off.

1
Model Registry Query

Fetches model version, architecture, training parameters, and performance metrics from MLflow.

2
Data Catalog Query

Retrieves data source descriptions, lineage, versions, and processing steps from the data catalog.

3
Annex IV Template Population

Maps retrieved metadata to all required Annex IV fields. Flags missing information for human completion.

4
Draft Review & Export

Produces a structured draft document with completion percentage and list of items requiring human input.

Agent 3 â€” BiasWatchAgent Bias Monitoring

Runs on a weekly schedule to monitor live PulseCredit decisions for demographic bias. Automatically generates fairness reports and escalates if thresholds are breached.

What it does

BiasWatchAgent queries the pseudonymised PulseCredit decision log, computes fairness metrics across demographic groups, compares against established thresholds, and generates a structured report. If any metric breaches its threshold, it creates an incident ticket and notifies the Head of Data Science.
